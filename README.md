#CodePulse — 基于小米MiMo大模型的智能代码审查与重构Agent
##1. 项目概述
CodePulse 是一款面向 Python 开发者的智能代码审查与自动重构助手，深度集成小米 MiMo-V2.5 大模型。系统能够自动扫描项目中的 Python 代码，结合 MiMo 强大的代码理解与推理能力，生成结构化的质量审查报告，并针对具体问题提供可直接应用的重构方案。整个流程由多 Agent 协作驱动，支持并发处理海量文件，大幅降低人工 Code Review 成本，提升代码质量与工程效率。

##2. 项目背景与动机
在当前软件开发流程中，代码审查是保障质量的关键环节，但存在以下痛点：

人工审查耗时耗力，尤其在大规模项目中难以覆盖所有文件；

初级开发者容易遗漏安全隐患、性能瓶颈和不符合最佳实践的写法；

市面通用 AI 工具缺乏针对代码语义的深度分析，结果泛泛而谈。

小米 MiMo-V2.5 模型凭借100 万 Token 超长上下文、指令遵循与代码生成能力在 SWE-bench 开源赛道排名第一，特别适合处理复杂代码分析任务。本项目正是基于 MiMo 的这些核心优势，打造了一个能够自主完成“审查—诊断—重构”闭环的实用型 Agent，让开发者专注于高价值决策。

##3. 核心功能
功能模块	说明
多文件自动扫描	递归收集指定目录下所有 .py 文件，支持忽略虚拟环境等无关路径。
智能分块审查	长代码自动切割为适配模型上下文的块，并发调用 MiMo API 并行分析。
结构化报告输出	每个文件生成 Markdown 格式审查报告，包含严重问题、代码异味、性能建议、重构推荐四个维度。
重构代码生成	针对已识别的问题，由单独的重构 Agent 生成完整、可直接运行的优化后代码。
异步并发架构	基于 asyncio 实现多文件并行处理，充分利用 API 并发能力，缩短整体耗时。
##4. 技术架构
```text
┌───────────────┐    ┌───────────────┐    ┌───────────────┐
│   file_utils  │    │  api_client   │    │ report_gener. │
│ (文件收集/拆分) │    │ (MiMo API封装) │    │ (报告格式化)    │
└──────┬────────┘    └──────┬────────┘    └──────┬────────┘
       │                    │                    │
       └────────────────────┼────────────────────┘
                            ▼
                  ┌─────────────────┐
                  │    agent.py     │
                  │ (任务调度与控制)   │
                  └────────┬────────┘
                           │
                           ▼
                  ┌─────────────────┐
                  │ 小米MiMo-V2.5   │
                  │ (代码理解/推理)   │
                  └─────────────────┘
```
api_client.py：封装 MiMo API（完全兼容 OpenAI 格式），提供 review_code() 和 suggest_refactor() 两个核心接口，分别用于代码审查和重构建议生成。

file_utils.py：负责文件系统操作，包括递归查找 Python 文件、将长文件智能拆分为不超过 200 行的逻辑块，保障模型输入质量。

agent.py：主控制器，通过 CodeReviewAgent 类管理审查流程：遍历文件列表 → 异步提交审查任务 → 收集结果 → 调用报告生成器。

report_generator.py：将所有文件的审查结果汇总为一份结构清晰的 Markdown 报告，便于集成至 CI/CD 流程或直接阅读。

##5. 创新点与 MiMo 模型能力结合
多 Agent 协作流
系统内部分为“审查 Agent”和“重构 Agent”，前者负责发现问题，后者根据问题描述生成修复代码，形成 “发现→解决”的自动化闭环。这种链式协作极大释放了 MiMo 在代码生成与推理方面的一流能力。

超长上下文充分应用
MiMo-V2.5 支持 100 万 Token 上下文窗口。对于普通单体文件，可直接将完整代码作为输入；对于超大型源码库，我们采用智能分块策略，每块保留足够的上下文信息，确保审查连贯性。这与模型的原生长上下文优势完美匹配。

深度思考与结构化输出
通过在 system prompt 中定义严格的四维审查框架，引导 MiMo 进行有逻辑层次的推理，避免生成笼统评价。同时，重构 Agent 要求输出完整代码块，直接可用，减少了开发者二次修改的工作量。

实用场景聚焦
项目完全面向真实开发场景，已可在本地运行并输出可集成的审查报告。小米 Orbit 计划鼓励的 Agent 应用、编程工具 与本项目定位高度契合。

##6. 快速开始
###6.1 环境准备
Python 3.10+

小米 MiMo API Key（申请地址：https://platform.xiaomimimo.com/）

###6.2 安装依赖
bash
git clone https://github.com/your-org/code-pulse.git
cd code-pulse
pip install -r requirements.txt
###6.3 设置 API Key
bash
export MIMO_API_KEY="sk-xxxxxxxxxxxxxxxxxxxx"
export MIMO_BASE_URL="https://api.xiaomimimo.com/v1"   # 可选
###6.4 运行审查
bash
python agent.py /path/to/your/python/project
审查完成后，会在当前目录生成 code_review_report.md，内容涵盖所有 Python 文件的质量评估。
